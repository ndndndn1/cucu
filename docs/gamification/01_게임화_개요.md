# 01. 게임화 개요 — GPU Architect: Rise from Silicon

> **BareMetal-SGEMM 교육 과정을 위한 게임화 레이어 설계서**
> 본 문서는 기존 14개 교육 모듈(75개 퀴즈, 10개 시뮬레이션)의 콘텐츠를 변경하지 않고, 그 위에 동기부여 시스템을 덧씌우는 설계를 기술한다.

---

## 1. 세계관

### "GPU Architect: Rise from Silicon"

학생은 가상 반도체 회사 **SiliconForge**의 신입 GPU 아키텍트로 입사한다. 회사의 미션은 "세계에서 가장 빠른 행렬곱 엔진을 만드는 것"이다.

학생은 처음에 단순한 나이브 구현(~50 GFLOP/s)에서 시작하여, 하드웨어를 깊이 이해하고 최적화 기법을 마스터할 때마다 SiliconForge 내에서 승진한다. 최종 목표는 cuBLAS의 85-95%에 달하는 성능(~9500 GFLOP/s)을 달성하고, **칩 아키텍트**로 승진하여 전체 아키텍처를 설계할 수 있는 역량을 증명하는 것이다.

### 핵심 내러티브 요소

- **SiliconForge 본사**: 학습 대시보드가 가상 사무실/연구소 형태로 시각화
- **프로젝트명 "Titan"**: 차세대 GPU의 행렬곱 유닛 — 학생이 설계하고 최적화하는 대상
- **기술 보고서**: 각 모듈 완료 시 자동 생성되는 "SiliconForge 기술 노트" 형태의 학습 이력
- **승진 심사**: 마스터리 체크포인트(모듈 06, 08, 09)가 승진 심사 역할 — 80% 이상 통과 필수

## 2. 직급 체계

학습 진행에 따른 8단계 직급 체계. 각 직급은 실제 SGEMM 최적화 레벨의 성능 수치에 매핑된다.

| 단계 | 모듈 범위 | 직급 (한국어) | 직급 (영어) | 예상 GFLOP/s | cuBLAS 대비 | 해금 조건 |
|------|----------|-------------|------------|-------------|-----------|----------|
| 1 | 00-03 | 수습 엔지니어 | Intern Engineer | — | — | 과정 시작 |
| 2 | 04 (L0) | 주니어 엔지니어 | Junior Engineer | ~50-70 | 1-5% | 모듈 00-03 완료 |
| 3 | 05 (L1) | 엔지니어 | Engineer | ~200-300 | 5-15% | 모듈 04 완료 |
| 4 | 06 (L2) | 시니어 엔지니어 | Senior Engineer | ~800-1,200 | 20-40% | 모듈 05 완료 + 마스터리 체크포인트 1 통과 |
| 5 | 07 (L3) | 스태프 엔지니어 | Staff Engineer | ~2,000-3,000 | 50-70% | 모듈 06 완료 |
| 6 | 08 (L4) | 프린시펄 엔지니어 | Principal Engineer | ~4,000-6,000 | 70-85% | 모듈 07 완료 + 마스터리 체크포인트 2 통과 |
| 7 | 09 (L5) | 디스팅귀시드 엔지니어 | Distinguished Engineer | ~7,000-9,500 | 85-95% | 모듈 08 완료 + 최종 마스터리 체크포인트 통과 |
| 8 | 10 | 칩 아키텍트 | Chip Architect | 전체 마스터리 | — | 모듈 09-10 완료 + 포트폴리오 완성 |

### 직급별 특전

| 직급 | 해금되는 기능 |
|------|-------------|
| 수습 엔지니어 | 기본 학습 환경, FLOP 코인 획득 시작 |
| 주니어 엔지니어 | 학습 타임라인 열람, 용어 툴팁 구매 가능 |
| 엔지니어 | 레이더 차트 1축 해금 (하드웨어 이해) |
| 시니어 엔지니어 | 힌트 토큰 구매 가능, 레이더 차트 3축 해금 |
| 스태프 엔지니어 | 시뮬레이션 도우미 구매 가능, 코스메틱 상점 해금 |
| 프린시펄 엔지니어 | 멘토 모드 구매 가능 (SC 상점), 레이더 차트 전체 해금 |
| 디스팅귀시드 엔지니어 | 프리미엄 포트폴리오 테마 구매 가능, 리더보드 참여 가능 |
| 칩 아키텍트 | 전체 기능 해금, PDF 인증서 생성, JSON 내보내기 |

### 승진 연출

승진 시 다음 요소가 표시된다:

1. **승진 알림 배너**: "축하합니다! [이름]님이 **시니어 엔지니어**로 승진했습니다"
2. **성과 요약**: 해당 레벨에서 달성한 GFLOP/s, 정답률, 획득 FC/SC
3. **직급 배지 애니메이션**: 새 직급 배지가 프로필에 추가
4. **다음 직급 미리보기**: 다음 승진까지 필요한 조건 안내

## 3. 교육 이론과 게임화의 매핑

기존 교육 과정은 8가지 교육 이론을 적용하고 있다. 게임화 레이어는 이를 **강화**하되 **대체하지 않는다**.

| 교육 이론 | 기존 적용 | 게임화 보강 |
|----------|----------|-----------|
| **Bloom's Taxonomy** | 퀴즈 문항별 인지 수준 태깅 | 인지 수준별 차등 보상 (기억 10FC, 창조 30FC) |
| **인지 부하 이론** | 모듈당 신개념 3개 이하 | 미션 브리핑으로 학습 목표 가시화, 정보 과부하 방지 |
| **Worked Example** | L0-2 전체 워크스루 제공 | 코드 추적 게임으로 능동적 참여 유도 |
| **CRA 프레임워크** | 비유→다이어그램→수식 | 개념 매칭 드래그앤드롭으로 세 표현 간 연결 강화 |
| **숙달 학습** | 80% 게이트 퀴즈 | 마스터리 체크포인트를 "보스전"으로 극적 연출 |
| **이중 코딩** | ASCII 다이어그램 + 수식 | 인터랙티브 시뮬레이션이 시각적 요소를 조작 가능하게 확장 |
| **인출 연습** | 사전 복습 3문항 | 60초 타이머 기억력 챌린지로 게임화 |
| **정교화 질문** | "왜 작동하는가?" 질문 | "주니어에게 설명하기" 티치백 게임화 |

## 4. 게임화 시스템 전체 구조

```
┌─────────────────────────────────────────────────────┐
│                   학습자 (학생)                        │
│  ┌─────────┐  ┌──────────┐  ┌───────────────────┐  │
│  │ 직급 정보 │  │ 재화 지갑  │  │ 포트폴리오/이력     │  │
│  └────┬────┘  └────┬─────┘  └────────┬──────────┘  │
│       │            │                  │              │
├───────┴────────────┴──────────────────┴──────────────┤
│                  게임화 레이어                          │
│  ┌──────────┐  ┌───────────┐  ┌────────────────┐   │
│  │ 퀘스트체인 │  │ 보상 엔진  │  │ 업적/배지 시스템 │   │
│  │ (9섹션래퍼)│  │ (FC/SC)   │  │  (4 티어)       │   │
│  └────┬─────┘  └─────┬─────┘  └───────┬────────┘   │
│       │              │                 │             │
├───────┴──────────────┴─────────────────┴─────────────┤
│                기존 교육 콘텐츠 (변경 없음)               │
│  ┌─────────────────────────────────────────────────┐ │
│  │  14개 Markdown 모듈 · 75개 퀴즈 · 10개 시뮬레이션  │ │
│  │  3개 마스터리 체크포인트 · 50+ 용어 사전            │ │
│  └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 핵심 흐름

1. 학생이 모듈에 진입하면 **미션 브리핑**이 표시 (학습 목표 → 체크리스트)
2. 9개 섹션을 순서대로 진행하며, 각 섹션에 대응하는 **게임 래퍼**가 활성화
3. 게임 래퍼에서의 활동 결과가 **보상 엔진**에 전달 → FC/SC 적립
4. 모듈 완료 시 **배지** 수여, 조건 충족 시 **직급 승진**
5. 모든 활동은 **학습 타임라인**에 기록 → 포트폴리오에 반영

## 5. 설계 원칙

### 원칙 1: 게임화는 보강하되 대체하지 않는다

게임 요소는 기존 교육 콘텐츠 위에 얹히는 레이어이다. 게임화를 제거해도 교육적 가치는 100% 유지된다. 학생이 게임 요소 없이 순수하게 학습하는 것도 가능하다.

### 원칙 2: GFLOP/s를 통합 지표로 사용한다

실제 SGEMM 성능 수치(GFLOP/s)가 직급, 보상, 진행률의 기준이 된다. 이는 학생이 추상적인 "점수"가 아니라 실제 하드웨어 성능이라는 구체적 목표를 향해 학습하도록 한다.

### 원칙 3: 재화는 노력과 숙련도에 비례한다

FLOP 코인(FC)은 꾸준한 학습 활동에, 실리콘 크리스탈(SC)은 탁월한 성취에 비례한다. 시간만 투자한다고 무한히 쌓이지 않으며, 숙달도가 높을수록 보상 효율이 좋다.

### 원칙 4: 포트폴리오가 지속적 가치를 가진다

학습 이력은 단순한 게임 기록이 아니라, 실제 기술 역량을 증명하는 포트폴리오로 활용 가능하다. PDF 인증서와 JSON 데이터 내보내기를 지원하여 이력서나 LMS에 연동할 수 있다.

### 원칙 5: 한국어 우선

모든 게임화 요소의 기본 언어는 한국어이다. 기술 용어는 기존 교육 과정과 동일하게 한국어(영어) 병기를 유지한다. 예: "뱅크 충돌 (Bank Conflict)".

### 원칙 6: 실패는 학습 기회다

퀴즈 오답이나 시뮬레이션 실패에 대해 페널티를 부과하지 않는다. 대신 약점을 진단하고 복구 퀘스트를 제공한다. 재시도 시 보상이 감소하지만(20FC→10FC), 학습 기록이나 직급이 하락하지는 않는다.

### 원칙 7: 소셜 기능은 옵트인이다

리더보드, 코호트 비교 등 소셜 기능은 기본적으로 비활성이다. 학생이 명시적으로 선택한 경우에만 활성화된다. 기본 모드에서는 익명 퍼센타일 밴드("상위 20%에 위치")만 표시한다.

## 7. 기존 콘텐츠 참조 매핑

게임화 시스템이 참조하는 기존 교육 콘텐츠 현황:

| 기존 콘텐츠 | 수량 | 게임화 활용 |
|-----------|------|-----------|
| 교육 모듈 (`00`-`10`) | 11개 | 퀘스트 체인 11개로 변환 |
| 부록 모듈 (`11`-`13`) | 3개 | 용어 사전→툴팁 팩, 퀴즈 정답→채점 엔진, 시뮬레이션 가이드→인터랙티브 변환 |
| 퀴즈 문항 | 75개 (10개 모듈) | 퀴즈 게임 F 콘텐츠 |
| "왜 작동하는가" 질문 | ~40개 (모듈당 3-5개) | 티치백 게임 D 콘텐츠 |
| 사전 복습 질문 | ~30개 (모듈당 3개) | 기억력 챌린지 게임 A 콘텐츠 |
| 시뮬레이션 | 10개 | 인터랙티브 시뮬레이션 게임 E |
| 마스터리 체크포인트 | 3개 (모듈 06, 08, 09) | 보스전 3개 |
| 용어 사전 | 50+ 용어 | 용어 툴팁 팩 (FC 상점), 개념 매칭 게임 B |

> **참고**: 사전 복습 질문은 모듈 01-10에 존재하나, 모듈 00(서론)에는 없다. 따라서 기억력 챌린지는 모듈 01부터 활성화된다.
