# 04. 이력 및 포트폴리오

> 학습 타임라인, 성과 대시보드, 역량 레이더 차트, 업적 배지 시스템, 내보내기 기능 설계.

---

## 1. 학습 타임라인

### 1.1 개요

모든 학습 이벤트를 **Git 커밋 로그** 스타일로 시간순 기록한다. 학생은 자신의 학습 여정을 돌아보며 성장 과정을 확인할 수 있다.

### 1.2 기록 이벤트 유형

| 이벤트 유형 | 아이콘 | 기록 내용 |
|-----------|--------|---------|
| 모듈 시작 | 📂 | 모듈 번호, 시작 시각 |
| 섹션 완료 | ✅ | 섹션 번호, 게임 유형, 소요 시간 |
| 퀴즈 제출 | 📝 | 정답률, Bloom 분포, 획득 FC |
| 시뮬레이션 완료 | 🔬 | 정확도, 소요 시간, 획득 FC |
| 보스전 도전 | ⚔️ | 단계별 결과, 통과/실패, 획득 FC/SC |
| 직급 승진 | ⭐ | 이전 직급 → 새 직급, GFLOP/s |
| 업적 획득 | 🏆 | 배지 이름, 티어, 획득 조건 |
| 재화 변동 | 💰 | FC/SC 획득/소비, 잔액 |
| 상점 구매 | 🛒 | 아이템 이름, 가격 |
| 연속 접속 | 🔥 | 현재 streak 일수 |
| 모듈 완료 | 🎓 | 모듈 번호, 총 점수, 소요 시간, 배지 |

### 1.3 타임라인 표시 형식

```
[2025-01-15 14:23] 📂 모듈 04 (Level 0 — 나이브 구현) 시작
    └─ 직급: 주니어 엔지니어 | FC: 1,240

[2025-01-15 14:35] ✅ 섹션 2 완료 — 기억력 챌린지: 3/3 정답 (28초)
    └─ +50 FC (스피드 보너스 포함)

[2025-01-15 15:10] 🔬 시뮬레이션 4 완료 — 메모리 접근 패턴 시각화
    └─ 정확도: 92% | +100 FC | 소요: 18분

[2025-01-15 15:45] 📝 모듈 04 퀴즈 완료
    └─ 7/8 정답 (87.5%) | 콤보 최대: ×2.0 | +280 FC
    └─ Bloom 분포: 기억 2/2, 이해 2/2, 적용 2/2, 분석 1/2

[2025-01-15 15:45] 🎓 모듈 04 완료
    └─ 총 획득: 530 FC | 소요: 1시간 22분

    └─ 달성 GFLOP/s: ~200 | cuBLAS 대비: ~10%
```

### 1.4 타임라인 필터

- **이벤트 유형별**: 퀴즈만, 승진만, 재화 변동만 등
- **모듈별**: 특정 모듈의 이벤트만 표시
- **기간별**: 오늘, 이번 주, 이번 달, 전체

## 2. 성과 대시보드

### 2.1 GFLOP/s 진행 차트

학습 성과를 실제 커널 성능 수치에 매핑한 진행 그래프.

**개인 GFLOP/s 계산 공식**:

```
personal_gflops = base_gflops[level] × mastery_multiplier

base_gflops = {
    "수습":     0,
    "주니어":   50,
    "엔지니어": 200,
    "시니어":   800,
    "스태프":   2000,
    "프린시펄": 4000,
    "디스팅":   7000,
    "칩아키":   9500
}

mastery_multiplier = 0.5 + (quiz_accuracy × 0.5)
    # quiz_accuracy: 0.0 ~ 1.0 (해당 레벨 퀴즈 정답률)
    # mastery_multiplier 범위: 0.5 ~ 1.0
```

**차트 형태**:
- X축: 학습 진행 (모듈 번호 또는 날짜)
- Y축: GFLOP/s (로그 스케일, 50 ~ 10,000)
- 라인: 개인 GFLOP/s 곡선
- 참조선: cuBLAS 기준선 (약 10,000 GFLOP/s)
- 마일스톤 마커: 직급 승진 지점

### 2.2 모듈 마스터리 히트맵

모듈별 숙달도를 색상으로 표시한 그리드.

| 색상 | 숙달도 | 기준 |
|------|--------|------|
| 회색 | 미시작 | 모듈 미진입 |
| 빨강 | 시작됨 | 진행 중 (퀴즈 미완료) |
| 주황 | 통과 | 퀴즈 60-79% |
| 노랑 | 양호 | 퀴즈 80-89% |
| 초록 | 우수 | 퀴즈 90-99% |
| 금색 | 완벽 | 퀴즈 100% + 시뮬레이션 95%+ |

**표시 형태**: 11개 모듈(00-10)을 가로 1열로 배치

```
[00] [01] [02] [03] [04] [05] [06] [07] [08] [09] [10]
 🟡   🟢   🟢   🟡   🟢   🟠   ⬜   ⬜   ⬜   ⬜   ⬜
양호  우수  우수  양호  우수 통과  미시작 ...
```

### 2.3 핵심 통계 패널

| 통계 항목 | 설명 |
|----------|------|
| 현재 직급 | 이름 + 영문 + 배지 |
| 현재 GFLOP/s | 개인 계산 수치 |
| cuBLAS 대비 | 백분율 |
| 총 학습 시간 | 누적 활동 시간 |
| FC 잔액 / 총 획득량 | 현재 보유 / 역대 총합 |
| SC 잔액 / 총 획득량 | 현재 보유 / 역대 총합 |
| 퀴즈 평균 정답률 | 전체 75문항 기준 |
| 최장 연속 접속 | 일수 |
| 업적 수 | 획득 / 전체 |

## 3. 역량 레이더 차트

### 3.1 6축 구성

| 축 | 축 이름 | 관련 모듈 | 측정 기준 |
|----|--------|----------|---------|
| 1 | 하드웨어 이해 | 01, 02 | 모듈 01, 02 퀴즈 정답률 + 시뮬레이션 1, 2 정확도 |
| 2 | 메모리 최적화 | 04, 05, 06 | 모듈 04-06 퀴즈 정답률 + 시뮬레이션 4, 5, 6 정확도 + 보스전1 |
| 3 | 연산 최적화 | 07 | 모듈 07 퀴즈 정답률 + 시뮬레이션 7 정확도 + "왜 작동하는가" 점수 |
| 4 | 파이프라인 설계 | 08, 09 | 모듈 08-09 퀴즈 정답률 + 시뮬레이션 8, 9 정확도 + 보스전 2, 3 |
| 5 | 성능 분석 | 03, 10 | 모듈 03, 10 퀴즈 정답률 + 시뮬레이션 3, 10 정확도 |
| 6 | 수학적 기반 | 03, 07, 10 | Bloom "적용"+"분석" 레벨 문항 정답률 (해당 모듈) |

### 3.2 축별 점수 계산

각 축의 점수는 0-100 범위로 정규화된다.

```
axis_score = (
    quiz_accuracy × 0.4       # 퀴즈 정답률 (0-1)
    + simulation_accuracy × 0.3  # 시뮬레이션 정확도 (0-1)
    + bonus_score × 0.3         # 보너스 (보스전, "왜 작동하는가" 등)
) × 100
```

보너스 점수 구성:
- 보스전 통과: 0.5 (통과), 1.0 (만점)
- "왜 작동하는가" 답변률: 통과 비율
- 해당 축에 보스전이 없는 경우, 보너스 가중치를 퀴즈/시뮬레이션에 재분배

### 3.3 해금 진행

레이더 차트는 직급에 따라 점진적으로 해금된다:

| 직급 | 해금 축 |
|------|--------|
| 수습 엔지니어 | — (차트 미표시) |
| 엔지니어 | 하드웨어 이해 (1축) |
| 시니어 엔지니어 | + 메모리 최적화, 성능 분석 (3축) |
| 프린시펄 엔지니어 | 전체 6축 해금 |

### 3.4 시각화

- 6각형 레이더 차트 (Chart.js radar chart)
- 각 축에 점수 수치 표시
- 배경에 "목표 영역" 표시 (80점 이상 = 숙달 영역)
- 이전 측정값과 현재 값 오버레이 (성장 시각화)

조건이 사전에 공개되지 않는 특수 배지. 획득 후에만 조건이 표시된다.

| 배지 | 숨겨진 조건 | SC 보상 |
|------|-----------|--------|
| The Humble Beginning | 모듈 04에서 나이브 커널의 성능(~50 GFLOP/s)을 정확히 예측 | 2 |
| 100x | Level 0 → Level 5의 성능 향상 비율(100x+)을 정확히 계산하는 퀴즈 정답 | 2 |

### 4.6 배지 표시

- 각 배지는 **이름**, **아이콘**, **설명**, **획득 일시**, **티어 테두리**로 구성
- 프로필 페이지에 배지 컬렉션 그리드 표시
- 미획득 배지는 실루엣으로 표시 (히든 배지 제외 — 히든은 획득 전에는 보이지 않음)
- 최대 3개의 "대표 배지"를 프로필에 핀 가능

## 5. 내보내기

### 5.1 PDF 인증서

**수여 조건**: 칩 아키텍트 직급 달성 (전 과정 완료)

**인증서 구성 요소**:

| 영역 | 내용 |
|------|------|
| 상단 | SiliconForge 로고, "Certificate of Completion" |
| 수여자 | 이름, 날짜 |
| 직급 | "칩 아키텍트 (Chip Architect)" 배지 |
| 레이더 차트 | 6축 역량 차트 이미지 |
| 핵심 통계 | 총 학습 시간, 퀴즈 평균 정답률, 최고 GFLOP/s, 업적 수 |
| 하단 | 고유 인증 번호, QR코드 (검증용) |

**디자인 변형**:
- 기본 테마: 무료 (칩 아키텍트 달성 시 자동)
- 프리미엄 테마 3종: SC 상점에서 구매 (각 15 SC)
