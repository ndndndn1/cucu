# 05. 동기 설계

> 단기/중기/장기 동기 루프, 회복 메커니즘, 리더보드 설계.

---

## 1. 동기 루프 개요

게임화 시스템은 세 가지 시간 스케일의 동기 루프를 조합하여 학생의 지속적 참여를 유도한다.

```
장기 열망 (주~월)
  ├── 중기 목표 (일~주)
  │     ├── 단기 루프 (분~시간)
  │     │     ├── 퀴즈 콤보
  │     │     ├── 시뮬레이션 즉시 보상
  │     │     └── 일일 퀘스트
  │     ├── 모듈 완료 배지
  │     ├── 보스전
  │     └── 주간 연속 접속
  ├── 전 과정 완료
  ├── 포트폴리오 완성
  └── 월간 연속 접속
```

---

## 2. 단기 루프 (분~시간)

즉각적인 만족감과 미시적 진행 감각을 제공한다.

### 2.1 퀴즈 콤보 배율

| 연속 정답 | 배율 | 효과 |
|----------|------|------|
| 1-2 | ×1.0 | 기본 |
| 3-4 | ×1.5 | "좋은 시작!" 메시지 |
| 5-6 | ×2.0 | 화면 테두리 글로우 효과 |
| 7+ | ×3.0 | "불꽃 모드!" + 특수 사운드 |

콤보 리셋 시: "괜찮아요, 다시 시작!" 격려 메시지 + 관련 개념 복습 링크

### 2.2 시뮬레이션 즉시 보상

시뮬레이션 각 단계 완료 시 즉시 FC가 표시되는 "코인 드롭" 애니메이션.

| 정확도 | FC | 시각 효과 |
|--------|-----|---------|
| ~50% | 50 | 기본 코인 아이콘 |
| 80%+ | 100 | 은색 코인 + 반짝임 |
| 95%+ | 150 | 금색 코인 + 폭죽 |

### 2.3 일일 퀘스트

매일 3개의 미니 퀘스트가 생성된다. 학습 진행 상황에 맞춰 적응적으로 출제.

| 퀘스트 유형 | 내용 | FC 보상 |
|-----------|------|--------|
| 용어 매칭 | 임의 용어 3개의 한/영 매칭 | 15 |
| 스피드 계산 | FLOP 또는 대역폭 계산 1문제 (60초 제한) | 20 |
| 복습 문제 | 완료한 모듈에서 임의 퀴즈 1문항 | 15 |

**일일 퀘스트 규칙**:
- 매일 자정에 갱신
- 3개 모두 완료 시 추가 보너스 20 FC
- 완료하지 못한 퀘스트는 소멸 (누적 없음)
- 최소 1개 모듈을 완료해야 일일 퀘스트 활성화

## 3. 중기 목표 (일~주)

한 주 단위의 성취감과 방향성을 제공한다.

### 3.1 모듈 완료 배지

각 모듈 완료 시 Tier 1 배지가 수여된다. 배지 수여 시 전용 애니메이션과 함께 다음 정보가 표시:

- 배지 이름 및 아이콘
- 해당 모듈 통계 요약 (정답률, 소요 시간, 획득 FC)
- 다음 모듈 미리보기

### 3.2 보스전 (마스터리 체크포인트)

3개 보스전(모듈 06, 08, 09)은 중기 목표의 핵심 이정표이다.

| 보스전 | 모듈 | 직급 승진 기준 |
|--------|------|-------------|
| 뱅크 충돌 수호자 | 06 | 엔지니어 → 시니어 엔지니어 |
| 레이턴시 드래곤 | 08 | 스태프 엔지니어 → 프린시펄 엔지니어 |
| 최종 벤치마크 | 09 | 프린시펄 엔지니어 → 디스팅귀시드 엔지니어 |

**보스전 사전 예고**:
- 보스전 1개 모듈 전부터 "다음 미션 인텔"에서 보스전 존재를 예고
- "보스전 준비 체크리스트" 제공: 관련 개념 복습 항목 3-5개

### 3.3 주간 연속 접속 보상

| 연속 일수 | 보상 |
|----------|------|
| 3일 | 30 FC |
| 5일 | 50 FC |
| 7일 | 100 FC + 1 SC |

**보상 주기**: 7일을 달성하면 카운터가 리셋되어 다시 3일부터 보상

### 3.4 주간 도전 과제

매주 월요일에 갱신되는 특별 도전 과제 1개.

| 유형 | 예시 | 보상 |
|------|------|------|
| 시뮬레이션 마라톤 | 이번 주에 시뮬레이션 3개 완료 | 100 FC |
| 퀴즈 달인 | 이번 주 퀴즈 정답률 90% 이상 유지 | 150 FC |
| 꾸준한 학습자 | 이번 주 매일 최소 1섹션 완료 | 100 FC + 1 SC |
| Bloom 탐험가 | Bloom 4개 이상 레벨에서 정답 기록 | 80 FC |

## 4. 장기 열망 (주~월)

과정 전체를 관통하는 큰 그림과 지속적 동기를 제공한다.

### 4.1 전 과정 완료 — "Full Architect" 배지

| 조건 | 보상 |
|------|------|
| 모듈 00-10 전체 완료 | 500 FC + 50 SC + "Full Architect" Tier 3 배지 |
| 모든 마스터리 체크포인트 통과 | 포함 (위 조건에 포함) |
| 칩 아키텍트 직급 달성 | PDF 인증서 생성 해금 |

### 4.2 월간 연속 접속

| 연속 일수 | 보상 |
|----------|------|
| 14일 | 200 FC + 2 SC |
| 21일 | 300 FC + 3 SC |
| 30일 | 500 FC + 5 SC + "Streak Legend" Tier 3 배지 (고유) |

### 4.3 포트폴리오 완성

포트폴리오의 모든 구성 요소가 채워지면 특별 보상:

| 구성 요소 | 완성 조건 |
|----------|---------|
| 레이더 차트 6축 모두 70+ | 모든 영역 양호 이상 |
| 모듈 히트맵 전체 초록+ | 모든 모듈 90% 이상 |
| 업적 배지 20개+ | 33개 중 20개 이상 획득 |
| 학습 타임라인 100개+ 이벤트 | 충분한 학습 이력 |

**완성 보상**: 200 FC + 프리미엄 포트폴리오 프레임 (코스메틱)

### 4.4 GFLOP/s 마일스톤

| 마일스톤 | 개인 GFLOP/s | 보상 |
|---------|-------------|------|
| 첫 100 | 100+ | 50 FC + "The Humble Beginning" 히든 배지 기회 |
| 1,000 돌파 | 1,000+ | 100 FC |
| 5,000 돌파 | 5,000+ | 200 FC + "GFLOP/s Champion" Tier 3 배지 |
| 9,000 도달 | 9,000+ | 300 FC + 칭호 "TFLOP/s를 향하여" |

## 5. 회복 메커니즘

학습 이탈과 실패에 대한 회복 경로를 설계하여, 학생이 좌절 없이 학습을 지속할 수 있도록 한다.

### 5.1 단기 이탈 (1-2일)

**프리즈 토큰**으로 연속 접속 기록을 보존할 수 있다.

| 획득 방법 | 수량 |
|----------|------|
| 마스터리 체크포인트 통과 시 자동 지급 | 1개 |
| FC 상점에서 구매 | 100 FC / 1개 |
| 직급 승진 시 자동 지급 | 1개 |

**사용**: 이탈한 날에 자동 적용 (보유 토큰이 있는 경우)

### 5.2 중기 이탈 (3-6일)

- 연속 접속 기록은 리셋되나, **학습 진행률과 재화는 보존**
- 복귀 시 "환영합니다" 메시지 + 마지막 활동 요약

### 5.3 장기 이탈 (7일+)

"환영합니다" 퀘스트 체인이 활성화된다:

| 단계 | 내용 | 보상 |
|------|------|------|
| 1 | 진행 요약 표시 — 현재 직급, 마지막 완료 모듈, 보유 재화 | — |
| 2 | 복습 퀴즈 3문항 (마지막 완료 모듈에서 출제) | 30 FC |
| 3 | 다음 학습 목표 안내 + 동기부여 메시지 | 20 FC |

**핵심 원칙**: 장기 이탈에 대한 **페널티가 없다**. 학습 기록, 재화, 업적은 모두 보존된다. 오직 연속 접속 기록만 리셋된다.

### 5.4 퀴즈 실패 회복

퀴즈에서 60% 미만을 기록하면 약점 진단 및 복구 퀘스트가 제공된다.

**진단 과정**:
1. Bloom 레벨별 정답률 분석
2. 가장 약한 Bloom 레벨 식별 (예: "분석" 레벨 0/2)
3. 약점 영역에 맞는 복구 퀘스트 생성

**복구 퀘스트**:

| Bloom 레벨 | 복구 활동 | 문항 수 | 보상 |
|-----------|---------|--------|------|
| 기억 | 용어 플래시카드 복습 | 5 | 30 FC |
| 이해 | 개념 매칭 재도전 | 3 | 30 FC |
| 적용 | 코드 추적 간소화 버전 | 3 | 40 FC |
| 분석 | 비교 질문 (A vs B 방식의 차이) | 3 | 40 FC |
| 평가 | "왜 작동하는가" 재도전 (힌트 포함) | 2 | 50 FC |
| 창조 | 간단한 설계 변형 질문 | 2 | 50 FC |

### 5.5 반복 실패 (동일 모듈 3회+ 실패)

학생이 같은 모듈 퀴즈를 3회 이상 통과하지 못한 경우:

| 단계 | 지원 내용 |
|------|---------|
| 3회 실패 | 힌트 비용 할인: 50 FC → 30 FC |
| 4회 실패 | 단계적 스캐폴딩: 관련 섹션 요약 자동 표시 |
| 5회 실패 | 개념 설명 섹션 재방문 권장 + 시뮬레이션 재도전 권장 |

**핵심**: 실패 횟수에 따라 **지원은 증가**하지만, **페널티나 잠금은 없다**.

### 5.6 마스터리 체크포인트 실패

보스전(마스터리 체크포인트)에서 80% 미만인 경우:

1. **단계별 피드백**: 어떤 단계에서 실패했는지 상세 분석
2. **관련 섹션 링크**: 해당 개념이 설명된 모듈 섹션으로 바로 이동
3. **무제한 재시도**: 횟수 제한 없이 재도전 가능
4. **FC 보상 유지**: 재시도에서도 통과 시 300 FC 동일 지급
5. **SC 차이**: SC는 1차 시도 만점에만 지급 (재시도에서는 FC만)

## 6. 리더보드

### 6.1 기본 모드 (옵트인 불필요)

모든 학생에게 기본 제공되는 **익명 비교 기능**:

```
당신의 위치: 상위 15%
    ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░
    하위 ──────────────── 상위
                        ▲ 당신
```

- GFLOP/s 기준 퍼센타일 밴드 표시
- 다른 학생의 개별 정보는 표시하지 않음
- 집계 데이터만 사용 (백엔드에 익명 통계 전송)

### 6.2 경쟁 리더보드 (옵트인)

학생이 명시적으로 참여를 선택한 경우에만 활성화.

| 리더보드 | 기준 | 갱신 |
|---------|------|------|
| GFLOP/s 랭킹 | 개인 GFLOP/s 점수 | 실시간 |
| 완료 속도 | 최초 칩 아키텍트 달성 시간 | 실시간 |
| 연속 기록 | 현재 연속 접속 일수 | 일 1회 |
| 주간 FC 획득량 | 이번 주 획득한 FC | 주 1회 리셋 |

### 6.3 클래스/코호트 리더보드

대학 수업이나 스터디 그룹에서 사용할 수 있는 그룹 리더보드.

- **코호트 생성**: 강사가 초대 코드를 생성하여 학생들에게 배포
- **코호트 통계**: 평균 진행률, 평균 GFLOP/s, 완료율
- **코호트 리더보드**: 코호트 내 순위 (옵트인 기반)
- **강사 대시보드**: 학생별 진행 현황, 공통 약점 영역 분석

### 6.4 리더보드 안전장치

| 정책 | 내용 |
|------|------|
| 옵트인 전용 | 기본적으로 경쟁 리더보드 비활성 |
| 닉네임 사용 | 실명 대신 닉네임 표시 가능 |
| 언제든 탈퇴 | 리더보드 참여를 언제든 해제 가능 (데이터 즉시 삭제) |
| 상위만 표시 | 상위 20명만 표시, 하위 순위는 표시하지 않음 |
| 반칙 방지 | 비정상 패턴 감지 시 리더보드에서 자동 제외 |

## 7. 알림 설계

### 7.1 인앱 알림

| 알림 유형 | 트리거 | 우선순위 |
|----------|--------|---------|
| 직급 승진 | 승진 조건 충족 | 높음 (모달) |
| 배지 획득 | 배지 조건 충족 | 높음 (토스트) |
| 보스전 해금 | 이전 모듈 완료 | 중간 (배너) |
| 일일 퀘스트 갱신 | 매일 자정 | 낮음 (뱃지) |
| 연속 접속 위험 | 23시간 미접속 시 | 중간 (푸시, 옵트인) |
| 주간 도전 갱신 | 매주 월요일 | 낮음 (뱃지) |

### 7.2 알림 빈도 제한

- 하루 최대 5개 알림
- 조용한 시간 설정 가능 (예: 23:00-08:00 알림 차단)
- 모든 알림 유형별 개별 끄기 가능
